🧠 MACHINE LEARNING
════════════════════════════════════════════════════════════════════════════

Algorithm: Motion-Based Point Detection

  Step 1: Extract video frames
  Step 2: Analyze frame sequences (30-frame buffer)
  Step 3: Calculate optical flow/motion between frames
  Step 4: Score motion level
  Step 5: Classify as "point" (high motion) or "break" (low motion)
  Step 6: Generate segment timestamps

Performance:
  └─ Speed: 30-120 seconds for 5-10 min video
  └─ Memory: 200-500 MB during processing
  └─ Accuracy: Works best with clear camera angles

Tunable Parameters:
  └─ motion_threshold = 5000 (point detection sensitivity)
  └─ still_threshold = 2000 (break detection sensitivity)
  └─ buffer_size = 30 (frame analysis window)

❓ TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════

Backend won't start:
  → Check Python version: python3 --version
  → Reinstall deps: pip install --upgrade -r requirements.txt

Port 5000 in use:
  → Change port in backend.py and frontend.html

Videos not processing:
  → Check backend console for errors
  → Verify video codec is supported
  → Ensure disk space available

Poor ML detection:
  → Adjust motion_threshold in backend.py
  → Test with well-lit videos first
  → Check for clear camera movement
